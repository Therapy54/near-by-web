# Near-By Skill System

This directory contains the authoritative rules, conventions, and architectural protocols for the Near-By project. These documents are designed to guide both human developers and AI agents to ensure project consistency and stability over time.

## Files

| File | Purpose |
|---|---|
| `coding_standards.md` | Mandatory variable declaration and formatting rules |
| `architecture.md` | Context-Based directory structure and isolation rules |
| `verification_process.md` | testing, verification, build and documentation requirements before any code is merged |
| `agent_protocol.md` | Mandatory operating instructions for any AI agent |
| `api-contracts/SKILL.md` | API contract system for Near-By including endpoints, request/response between frontend and backend. |

## Critical Rule

> **No code change is valid until it passes the verification process.**  
> No architectural deviation is allowed without updating these documents.

Every developer and AI agent MUST read all files in this directory before making any change to the codebase.
